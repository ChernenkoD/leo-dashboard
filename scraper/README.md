# Автообновление данных из LEO

Скрипт раз в день (2 раза, по расписанию в GitHub Actions) заходит в LEO
под твоей сохранённой сессией и обновляет `data.json` в этом репозитории.

## Разовая настройка (сделай это сам — пароль я не трогаю)

1. Установи зависимости локально:
   ```
   cd scraper
   pip install -r requirements.txt
   playwright install chromium
   ```

2. Запусти скрипт логина — откроется окно браузера:
   ```
   python3 login_and_save_session.py
   ```
   Залогинься в LEO своими данными, вернись в терминал, нажми Enter.
   Появится файл `storage_state.json`.

3. Залей сессию в секрет GitHub (сам, в терминале):
   ```
   gh secret set LEO_SESSION --repo ChernenkoD/leo-dashboard < storage_state.json
   ```

4. Проверь, что всё работает — запусти workflow вручную:
   ```
   gh workflow run scrape-leo.yml --repo ChernenkoD/leo-dashboard
   ```
   Через минуту-две глянь:
   ```
   gh run list --repo ChernenkoD/leo-dashboard --limit 1
   ```

## Когда сессия протухнет

Workflow в логах напишет "Сессия LEO протухла". Тогда просто повторяешь
шаги 2–3 ещё раз — это 2 минуты.
